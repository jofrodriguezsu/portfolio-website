JWT Authentication with Refresh Tokens

I built a JWT authentication system using Node.js and Express, where users can register and log in with securely hashed passwords using bcrypt. The app uses access tokens and refresh tokens to handle authentication, protects private routes with authorization middleware, and allows users to refresh their session without logging in again. For simplicity, it uses an in-memory data store to focus on the authentication flow and best practices, with configuration managed through dotenv.
