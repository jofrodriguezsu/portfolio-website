Book Management API

Built this RESTful API to get hands-on experience managing data with Node.js, Express, and MongoDB. I used an MVC (Model-View-Controller) structure to keep the code organized and easy to maintain . To make the app more reliable, I added Mongoose schema validation (like making sure a book's "year" isn't in the future) and built a custom error handler to manage any issues smoothly . I also used environment variables to keep sensitive database credentials secure.

In-Memory RESTful User API

This project is a simple backend application built with Node.js and Express. It provides a RESTful API for managing users, supporting basic CRUD operations (Create, Read, Update, and Delete).

User data is stored in memory, allowing me to focus on understanding API logic, routing, and HTTP methods such as GET, POST, PUT, and DELETE, as well as core REST principles like stateless requests and clean endpoint design.
